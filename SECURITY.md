# Security Policy

We take security seriously. Thank you for helping keep AgriOS and its users
safe.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 3.x     | Yes                |
| 2.x     | Critical fixes only |
| < 2.0   | No                 |

## Reporting a Vulnerability

**Please do not report security issues through public GitHub issues.**

Instead, email `security@2africa.ai` with:

- A description of the issue
- Steps to reproduce
- The version affected
- Any potential impact you've identified
- Optional: a suggested fix

If you prefer encrypted communication, you can use our PGP key (fingerprint
to be published at https://2africa.ai/security).

## Our Commitments

When you report a vulnerability, we promise to:

1. **Acknowledge** receipt within 3 business days
2. **Investigate** and confirm the issue within 7 business days
3. **Keep you informed** of remediation progress
4. **Credit** you publicly (or anonymously, your choice) when we publish a fix

## Disclosure Timeline

- Day 0: report received
- Day 0-7: we triage and confirm
- Day 7-30: we develop and test a fix
- Day 30: coordinated public disclosure with patched release

For critical issues we may accelerate this timeline. For low-severity issues
we may bundle the fix into the next regular release.

## Out of Scope

- Vulnerabilities in third-party dependencies (please report to upstream)
- Social engineering of project maintainers
- Physical access to a deployed AgriOS instance
- Issues that require an attacker to already have admin access

## Safe Harbor

We will not pursue legal action against researchers who:

- Follow this policy
- Make a good-faith effort to avoid privacy violations and service disruption
- Do not access more data than necessary to demonstrate the vulnerability
- Give us reasonable time to fix the issue before public disclosure

Thank you for helping protect African agriculture infrastructure.
